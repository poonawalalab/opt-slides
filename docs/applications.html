Forward & Inverse Kinematics

Given joint angles ùê™={qi}\mathbf q = \{q_i\} we can predict the end effector pose ùê±=(x,y,Œ∏)\mathbf x = (x,y,\theta)

Forward & Inverse Kinematics

  • The Forward Kinematics problem ùê±=f(ùê™)\mathbf x = f(\mathbf q) combines known closed-form expressions for individual homogenous transformations
  • Computing the inverse ùê™=f‚àí1(ùê±)\mathbf q = f^{-1}(\mathbf x), however, is not as easy
  • The inverse kinematics problem is often not even unique, which has algorithmic implications

IK Approaches

Since we know how to build f(ùê™)f(\mathbf q), we arrive at two approaches to inverse kinematics

  • Analytic approaches: Build the closed-form expression f(ùê™)f(\mathbf q) and define a closed-form inverse f‚àí1(ùê™)f^{-1}(\mathbf q)
  • Numerical approaches: Numerically search for values of ùê™\mathbf q so that f(ùê™)=ùê±f(\mathbf q) = \mathbf x, where the function ff is either closed-form or numerical

Numerical IK Approach

  • solve optimization: minùê™‚Äñùê±‚àíf(ùê™)‚Äñ22\min_{\mathbf q}\quad \lVert \mathbf x - f(\mathbf q) \rVert_2^2
  • We can add constraints that make the solution unique, or other benefits

Logistic Regression

  • We have vectors ùêöi‚àà‚Ñùd\bm{a}_i \in \mathbb{R}^d for i=1,2,‚Ä¶,n1i = 1, 2, \ldots, n_1 in a class and vectors ùêõj‚àà‚Ñùd\bm{b}_j \in \mathbb{R}^d for j=1,2,‚Ä¶,n2j = 1, 2, \ldots, n_2 not in that class.

Two clusters of points ùêöi\bm{a}_ {i} (red) and ùêõj\bm{b}_ {j} (green) in ‚Ñù2\mathbb R^2 (d=2)

Approach

  • We wish to classify the points into one of two clusters

  • We convert classification into regression by requiring that for some function ff: f(ùêöi)=1 and f(ùêöi)=0 f(\bm{a}_i) = 1 \text{ and } f(\bm{a}_i) = 0

  • One solution is to use the logistic function after linearly mapping inputs ùê±\bm{x} to a scalar: logistic(s)=es1+es;s(ùê±)=ùê±Tùê∞+Œ≤ \mathrm{logistic}(s) = \frac{e^s}{1+e^{s}}; \quad s(\bm{x}) = \bm{x}^T \bm{w} + \beta

  • New goal: find a vector ùê∞‚àà‚Ñùd\bm{w} \in \mathbb{R}^d and a number Œ≤\beta such that

exp(ùêöi‚ä§ùê∞+Œ≤)1+exp(ùêöi‚ä§ùê∞+Œ≤)‚âà1,‚àÄi, and exp(ùêõj‚ä§ùê∞+Œ≤)1+exp(ùêõj‚ä§ùê∞+Œ≤)‚âà0,‚àÄj. \frac{\operatorname{exp}(\bm{a}_i^\top \bm{w} + \beta)}{1 + \operatorname{exp}(\bm{a}_i^\top \bm{w} + \beta)} \approx 1, \;\; \forall i, \quad \text{ and } \quad \frac{\operatorname{exp}(\bm{b}_j^\top \bm{w} + \beta)}{1 + \operatorname{exp}(\bm{b}_j^\top \bm{w} + \beta)} \approx 0, \;\; \forall j.

Approach

  • This problem can be cast as an unconstrained optimization problem

maximizeùê∞,Œ≤(‚àèiexp(ùêöi‚ä§ùê∞+Œ≤)1+exp(ùêöi‚ä§ùê∞+Œ≤))(‚àèj(1‚àíexp(ùêõj‚ä§ùê∞+Œ≤)1+exp(ùêõj‚ä§ùê∞+Œ≤))) \operatorname{maximize}_{\bm{w}, \beta} \left(\prod_i \frac{\operatorname{exp}(\bm{a}_i^\top \bm{w} + \beta)}{1 + \operatorname{exp}(\bm{a}_i^\top \bm{w} + \beta)}\right) \left(\prod_j \left(1 - \frac{\operatorname{exp}(\bm{b}_j^\top \bm{w} + \beta)}{1 + \operatorname{exp}(\bm{b}_j^\top \bm{w} + \beta)} \right) \right)

which may equivalently be expressed using a log transformation as

minimizeùê∞,Œ≤‚àëilog(1+exp(‚àíùêöi‚ä§ùê∞‚àíŒ≤))+‚àëjlog(1+exp(ùêõi‚ä§ùê∞+Œ≤)). \operatorname{minimize}_{\bm{w}, \beta} \sum_i \operatorname{log}\left(1 + \operatorname{exp}(-\bm{a}_i^\top \bm{w} - \beta) \right) + \sum_j \operatorname{log}\left(1 + \operatorname{exp}(\bm{b}_i^\top \bm{w} + \beta) \right).

‚àè(eùêöi‚ä§ùê∞+Œ≤1+eùêöi‚ä§ùê∞+Œ≤)=‚àè(11+e‚àíùêöi‚ä§ùê∞‚àíŒ≤) \prod \left( \frac{e^{\bm{a}_i^\top \bm{w} + \beta}}{1 + e^{\bm{a}_i^\top \bm{w} + \beta}} \right) = \prod \left( \frac{1}{1 + e^{-\bm{a}_i^\top \bm{w} - \beta}} \right)

‚àè(1‚àíeùêõj‚ä§ùê∞+Œ≤1+eùêõj‚ä§ùê∞+Œ≤)=‚àè(11+eùêõj‚ä§ùê∞+Œ≤) \prod \left( 1 - \frac{e^{\bm{b}_j^\top \bm{w} + \beta}}{1 + e^{\bm{b}_j^\top \bm{w} + \beta}} \right) = \prod \left( \frac{1}{1 + e^{\bm{b}_j^\top \bm{w} + \beta}} \right)

# Generate a random non-trivial linear program.

np.random.seed(1)

plt.figure(figsize=(8, 6))

# c = -A.T @ lamb0

#                  [A @ x <= beta])
prob.solve()

# Print result.
print("\nThe optimal value is", prob.value)
print("A solution x is")

Finite Markov Decision Processes

  • A finite MDP, is described by the tuple (ùíÆ,ùíú,T,Œ≥,‚Ñõ)(\mathcal{S}, \mathcal{A}, T,\gamma,\mathcal{R}):
    • ùíÆ\mathcal S: A finite set of states
    • ùíú\mathcal A: A finite set of actions that can be taken in each state
    • TT: A (probabilistic) transition map
    • Œ≥\gamma: The discount factor
    • ‚Ñõ\mathcal R: the reward function
  • The agent and environment interact at each of a sequence of discrete time steps, t=0,1,2,‚Ä¶t = 0, 1, 2, \ldots.
    • At each time step tt, the agent receives some respresentation of the environment‚Äôs state, St‚ààùíÆS_t \in \mathcal{S}, and on that basis selects an action At‚ààùíú(s)A_t \in \mathcal{A}(s).
    • One time step later, in part as a consequence of its actions, the agent receives a numerical reward, Rt+1‚àà‚Ñõ‚àà‚ÑùR_{t+1} \in \mathcal{R} \in \mathbb{R} and finds itself in a new state, St+1S_{t+1}.
  • The MDP and agent together give rise to a trajectory that begins like this: S0,A0,S1,R1,A1,S2,R2,A2,R3,‚Ä¶(1) S_0, A_0, S_1, R_1, A_1, S_2, R_2, A_2, R_3, \ldots \qquad(1)

Transition Model

  • For finite MDPs, the random variables RtR_t and StS_t have well defined discrete probability distributions dependent on the preceding state and action.

T(s‚Ä≤,r‚à£s,a)‚âú‚Ñô{St=s‚Ä≤,Rt=r‚à£St‚àí1=s,At‚àí1=a}(2) T(s', r \mid s, a) \triangleq \mathbb{P}\{S_t = s', R_t = r \mid S_{t-1} = s, A_{t-1} = a\} \qquad(2)

  • This function TT defines the dynamics of the MDP.
    • It specifies a probability distribution over SS for each choice of ss and aa, i.e.,

‚àës‚Ä≤‚ààùíÆ‚àër‚àà‚ÑõT(s‚Ä≤,r‚à£s,a)=1,‚àÄs‚ààùíÆ,a‚ààùíú(s).(3) \sum_{s' \in \mathcal{S}} \sum_{r \in \mathcal{R}} T(s', r \mid s, a) = 1, \quad \forall s \in \mathcal{S}, \; a \in \mathcal{A}(s). \qquad(3)
